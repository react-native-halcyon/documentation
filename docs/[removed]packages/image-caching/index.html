Image caching

Using react-native-fast-image

I prefer to use react-native-fast-image for displaying images and caching them for later use

Get the package here :

https://js.coach/react-native-fast-image?search=fast-image&collection=React+Native

Preload local images for later use and for faster image loading

import { Image } from 'react-native';
import FastImage from 'react-native-fast-image';
export function preloadImages() {
  const images = [
    require('../assets/images/graphic1.webp'),
    require('../assets/images/graphic2.jpg'),
    require('../assets/images/graphic3.webp')
  ];
  const uris = images.map(image => ({
    uri: Image.resolveAssetSource(image).uri
  }));
  FastImage.preload(uris);
}

Usage

<FastImage
    style={{
        backgroundColor: '#ffffff',
        paddingTop: 2,
        paddingRight: 2,
        paddingLeft: 2,
        paddingBottom: 2,
        height: 170,
        borderColor: '#ecf0f1',
        justifyContent: 'space-between',
    }}
    source={require('./assets/images/logo.png')}
    resizeMode={FastImage.resizeMode.contain}
/>

Importing all images in one file

one stop image declaration and to easily cache them without changing the code from App.js

/* eslint-disable global-require */
const IMAGES = {
  'mimosagolf.png': require('./app_icons/mimosagolf.png'),
  '1-nologo.gif': require('./app_icons/1-nologo.gif'),
  'left-arrow.png': require('./app_icons/left-arrow.png'),
  'united-states.png': require('./app_icons/united-states.png'),
  'login_background.jpg': require('./large_images/login_background.jpg'),
  'rain-forest.png': require('./large_images/rain-forest.png'),
  'daintree-rainforest.jpg': require('./large_images/daintree-rainforest.jpg'),
  'abandoned-plane.jpeg': require('./large_images/abandoned-plane.jpeg'),

  'glazed_duck_fillet.jpg': require('./large_images/restaurant/glazed_duck_fillet.jpg'),

Usage

import IMAGES from "./assets/images";

<FastImage
  source={IMAGES["glazed_duck_fillet.jpg"]}
  style={styles.featuredItem}
  resizeMode={FastImage.resizeMode.cover}
/>
